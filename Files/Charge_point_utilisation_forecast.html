Charge point utilisation forecast¶

This notebook contains a simple version of a machine learning (ML) model that uses traffic data to estimate KWh consumption at charge point. The data used was synthetically generated.

Introduction¶

The cell below imports the synthetic data. This data set is made by the following columns

  • kWh which represents the energy consumption;
  • total traffic; and

  • segment X which represent traffic segments.

    A traffic segment is a portion of the traffic that is determined by certain characteristics. For example, segment 1 could represent trips that have travelled for at least 100 km before they traversed the charge point vicinity, have another 100 km to reach destination, are more than 80 km away from their residence and ara travelling to a non-work, non-home location.

    An important part of the ML exercise is determining what features (segments) contain more information about the charge point utilisation which is performed further below via two mehanism, mutual information regression and recursive feature elimination.

Train-Test Split¶

Before we look at the data any further let's split it between a test and a training set. The size of the data sample is not big and we want both the test and the train splits to be as representative as possible of the whole population of charge points. To achive this we introduce a new column indicating what quartile the site belongs to and use stratified sampling based on that.

df_initial['quartiles'] = pd.qcut(df_initial['kwh'], [0, .25, .5, .75, 1.])

A basic feature engineering protocol that should almost alwyas be applied is standard scaling. This process ensures that the scale of the feature has no impact on its relevancy.

std_scaler = StandardScaler()

As mentioned above feature selection is an important part of this process as it helps reduce the number of features used minimising the chances of overfitting or finding an arbitrary relationship between features and target that will not be present in the test data or the 'real world'.

To achieve this a mutual information regression analysis was performed to select the 15th most informative variables available.

k = 15
# define feature selection
fs = SelectKBest(k=k, score_func=mutual_info_regression)
# apply feature selection
fs = fs.fit(X, charge_points_labels)
s_cols = fs.get_feature_names_out(input_features=charge_points.columns)
X_r = fs.transform(X)

Both the standard scaler and the mutual information analysis were inserted into a pipeline for later use in the training data set.

features_pipeline = Pipeline([('std_scaler', std_scaler), ('selectkbest', fs)])

Focusing only on the selected models se can now fine tune the hyperparameters to find the best performing combination of these. Further more recursive feature elimination with a 5 fold cross-validation is performed to skim the features that don't add value to the model.

The scoring mechanism used will be mean absolute percentage error since we want to minimise the proportional error made giving small consumptions the same relative importance as possible high consumptions outliers.

We perform the search of the best parameters through a grid search with a 5 fold cross-validation.

param_dist = [{'alpha': list(range(1, 100))}]

estimator = long_list_models[short_list[0]]

scoring = 'neg_mean_absolute_percentage_error'

cv = 5

step = 1

def featureselection_gridsearch(estimator, param_dist, X, target, groups,
     step=step, cv=cv, scoring='neg_mean_absolute_percentage_error'):
    selector = RFECV(estimator, step=step, cv=cv,
                     scoring=scoring)
    selector = selector.fit(X, target)
    #
    X_rs = X[:, selector.support_]
    #
    grid_search = GridSearchCV(
        estimator, param_dist, cv=cv, scoring=scoring, return_train_score=True)
    grid_search.fit(X_rs, target, groups=groups)
    gs_best_score = grid_search.best_score_
    gs_best_estimator = grid_search.best_estimator_
    return selector.support_, gs_best_score, gs_best_estimator

param_dist = {'lasso': [{'alpha': list(range(1, 100))}],
              'sgdr': [{'loss': ['squared_error', 'huber', 'epsilon_insensitive', 'squared_epsilon_insensitive'],
                        'penalty': ['l2', 'l1', 'elasticnet'],
                        'alpha': [10**x/10**2 for x in range(0, 4)],
                        'l1_ratio': [0.1, 0.15, 0.2]}],
              'ridge': [{'alpha': [10**x/10**2 for x in range(0, 4)],
                         'fit_intercept': [False, True],
                         'normalize': [False, True],
                         'solver': ['auto', 'svd', 'cholesky', 'lsqr', 'sparse_cg', 'sag', 'saga', 'lbfgs'],
                         'positive': [False, True]}],
              'br': [{'alpha_1': [10**x/10**8 for x in range(0, 4)],
                      'alpha_2': [10**x/10**8 for x in range(0, 4)],
                      'lambda_1': [10**x/10**8 for x in range(0, 4)],
                      'lambda_2': [10**x/10**8 for x in range(0, 4)]}]}

gridsearch_results = {}
for m in short_list:
    estimator = long_list_models[m]
    selected_features, best_score, best_estimator = featureselection_gridsearch(estimator, param_dist[m], X=X_r, target=charge_points_labels,
                                                                                groups=charge_points_quartiles, step=step,
                                                                                cv=cv, scoring='neg_mean_absolute_percentage_error')
    gridsearch_results[m] = {'selected_features': selected_features,
                             'best_score': best_score,
                             'best_estimator': best_estimator}
    gridsearch_results[m]['full_pipeline'] = ColumnTransformer(
        [('features_pipeline', features_pipeline), gridsearch_results[m]['best_estimator']])

def reg(df, x, y, title=None):
    R2 = round(df[x].corr(df[y]), 2)
    slope, intercept, r_value, pv, se = stats.linregress(df[x], df[y])
    plt.figure(figsize=(16, 10))
    sns.regplot(data=df, x=x, y=y, line_kws={'label': '$y=%3.7s*x+%3.7s; R^2=%3.7s$' % (
        round(slope, 3), round(intercept, 3), round(r_value**2, 3))}).set(title=title)
    plt.legend()

Ensemble voting regressor¶

One last trick we can use to make the estimation more robust and potentially better is averaging the predictions of the algorithms hoping that the shortcomings the may have could be made up by the others. The results below don't show significant improvement in performance. Nevertheless, it's worth considering using to make the results more stable.

col_values= [(-gridsearch_results[m]['best_score'], gridsearch_results[m]['test_score']) for m in short_list]
col_values.append((np.nan, mean_absolute_percentage_error(
    test_cp_labels.values, df_test['average_prediction'])))
ind = list(short_list)

Training on full data set¶

Finally we can train on the whole data set save the model and get ready for deployment.

# train on full data
all_charge_points = df_initial.drop(columns=['kwh', 'quartiles'])
all_charge_points_labels = df_initial['kwh'].copy()
all_charge_points_quartiles = df_initial['quartiles'].copy()

all_cp_ready = features_pipeline.fit_transform(
    X=all_charge_points, y=all_charge_points_labels)

best_models_retrained = gridsearch_results.copy()

prediction = {}
for m in short_list:
    selected_features = best_models_retrained[m]['selected_features']
    best_models_retrained[m]['best_estimator'] = best_models_retrained[m]['best_estimator'].fit(
        all_cp_ready[:, selected_features], all_charge_points_labels.values)
    predictions = best_models_retrained[m]['best_estimator'].predict(all_cp_ready[:, selected_features])
    best_models_retrained[m]['best_score'] = mean_absolute_percentage_error(
        all_charge_points_labels.values, predictions)
    best_models_retrained[m]['full_pipeline'] = ColumnTransformer(
        [('features_pipeline', features_pipeline), best_models_retrained[m]['best_estimator']])
    prediction[m] = predictions
